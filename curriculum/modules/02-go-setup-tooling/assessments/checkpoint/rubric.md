# Module 02 Checkpoint Rubric

| Category | Weight | Meets standard |
|---|---:|---|
| Installation/environment | 15% | Explains version, PATH, and Go environment values. |
| Running/building | 15% | Separates Hello World, `go run`, and `go build` accurately. |
| Testing/formatting | 15% | Explains test compilation, assertions, and gofmt behavior. |
| Inspection tools | 10% | Explains `go vet`, `go doc`, and their limits. |
| Editor tooling | 10% | Explains `gopls` without replacing terminal proof. |
| Failure reading | 20% | Correctly distinguishes compiler, runtime, and test failures. |
| Module root/checklist | 15% | Explains `go.mod`, `go env GOMOD`, and a repeatable tooling checklist. |

## Passing standard

Pass: 80% or higher with no zero-score category.

Retake any category below 60%.

## Feedback guide

A strong answer:

- identifies what each command proves
- names the machine stage involved
- gives a first debugging move
- avoids treating green output as magic
- connects local commands to CI/release validation
