# Module 02 Checkpoint — Go Setup and Tooling

## Purpose

This checkpoint verifies that you can use the Go toolchain deliberately before starting programming fundamentals.

The assessment checks whether you can install, verify, run, build, test, format, inspect, and debug Go tooling output without treating the commands as magic.

## Scope

This checkpoint covers:

- Go installation verification
- `go version` and `go env`
- Hello World structure
- `go run`
- `go build`
- `go test`
- `gofmt`
- `go vet`
- `go doc`
- editor and `gopls` workflow
- compiler errors
- runtime errors
- test failures
- module root and `go.mod`
- tooling checklist

## Time expectation

Plan for 60 to 90 minutes.

## Allowed resources

You may use:

- Module 02 lesson READMEs
- terminal output
- official Go command help
- your editor
- your own notes

Do not use the answer key until you have attempted every question.

## Evidence required

Submit:

1. answers to `questions.md`
2. terminal transcript for `go version`, `go env GOMOD`, `go run`, `go test`, `gofmt -l`, and `go vet`
3. one compiler error explanation
4. one test failure explanation
5. one personal tooling checklist

## Passing standard

Pass with 80% or higher and no zero-score category.

Do not continue to Module 03 if you cannot explain the difference between run, build, test, format, vet, and module root.

## Retake policy

If a category is weak, return to the relevant lesson, run the commands again, revise your answer, and record the misunderstanding you corrected.
