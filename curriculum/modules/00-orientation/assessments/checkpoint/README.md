# Module 00 Checkpoint — Orientation

## Purpose

This checkpoint verifies that you understand how to use the Go Engineer curriculum before entering the technical modules.

The goal is not memorization. The goal is operational readiness: you can navigate the repository, understand the zero-magic contract, run examples, use starters correctly, and explain how proof works.

## Scope

This checkpoint covers:

- repository structure
- metadata vs learner-facing curriculum
- zero-magic learning
- lesson/lab/project/assessment workflow
- running code and tests
- starter and solution usage
- debugging question quality
- roadmap usage
- portfolio planning

## Time expectation

Plan for 45 to 60 minutes.

## Allowed resources

You may use:

- Module 00 lesson READMEs
- your terminal
- the repository files
- your own notes

You may not copy from the answer key before attempting the questions.

## Evidence required

Submit:

1. answers to `questions.md`
2. one terminal transcript showing a `go run` and `go test` command from Module 00
3. a short personal roadmap paragraph
4. a short portfolio plan paragraph

## Passing standard

You pass when your answers show that you can navigate the curriculum without guessing and can explain the proof workflow in your own words.

Use `rubric.md` to self-grade before comparing with `answer-key.md`.

## Retake policy

If you miss a category, return to the relevant lesson, revise your answer, and write one sentence explaining what changed in your understanding.
