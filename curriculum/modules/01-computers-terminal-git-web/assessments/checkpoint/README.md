# Module 01 Checkpoint — Computers, Terminal, Git, and the Web

## Purpose

This checkpoint verifies that the machine, terminal, Git, GitHub, and web vocabulary is safe enough for Go setup and programming work.

The goal is not to memorize commands. The goal is to explain what the system is doing and recover when something goes wrong.

## Scope

This assessment covers:

- programs, executables, and processes
- files, bytes, directories, and paths
- terminal basics
- environment variables
- exit codes
- process lifecycle
- memory preview
- Git snapshots, staging, commits, branches, and merges
- GitHub remotes and pull requests
- client/server, DNS, ports, and HTTP request/response

## Time expectation

Plan for 60 to 90 minutes.

## Allowed resources

You may use:

- Module 01 lesson READMEs
- your notes
- the terminal
- Git documentation built into your environment
- the repository files

Do not read the answer key until you have attempted the questions.

## Evidence required

Submit:

1. answers to `questions.md`
2. one terminal transcript showing `pwd`, `ls`, `go run`, and `go test`
3. one Git explanation diagram or text timeline
4. one HTTP request/response annotation
5. one paragraph describing your current debugging routine

## Passing standard

Pass with 80% or higher and no zero-score category.

A weak score in paths, terminal, or Git should be fixed before Module 02 because those weaknesses multiply quickly once Go tooling begins.

## Retake policy

If you miss a category, revisit the related lesson, revise the answer, and add a one-sentence explanation of the misunderstanding you corrected.
