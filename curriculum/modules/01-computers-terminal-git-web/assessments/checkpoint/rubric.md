# Module 01 Checkpoint Rubric

| Category | Weight | Meets standard |
|---|---:|---|
| Programs/processes | 15% | Correctly separates source, executable, process, OS ownership, and process lifecycle. |
| Files/terminal | 15% | Explains paths, working directory, shell behavior, and first debugging moves. |
| Environment/exit codes | 15% | Explains process configuration and success/failure signaling. |
| OS/memory preview | 10% | Gives beginner-safe explanations without overclaiming. |
| Git/GitHub | 20% | Explains snapshots, staging, commits, branches, merges, remotes, and PR review. |
| Web preview | 15% | Explains client/server, DNS, ports, and HTTP request/response vocabulary. |
| Reflection/debugging | 10% | Names a weak area and a concrete proof task to improve it. |

## Passing standard

Pass: 80% or higher with no category at zero.

Retake any category below 60%.

## Feedback guide

Strong answers:

- use concrete examples
- separate human mental model from machine view
- describe how to debug failure
- avoid treating commands as magic
- connect the concept to later Go work
