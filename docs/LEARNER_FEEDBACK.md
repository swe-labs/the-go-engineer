# Learner Feedback Loop

## Why

Technical correctness is only half of curriculum quality. Learner clarity and completion outcomes are also proof surfaces.

## How To Give Feedback

- Open a new issue using the docs or lesson template.
- Include lesson ID, what was unclear, and what output you expected.
- If possible, include the exact `RUN:` command and output snippet.

## Quick Link Template

Use this prefilled issue URL format in section READMEs:

`https://github.com/swe-labs/the-go-engineer/issues/new?template=docs.md&title=%5BDOCS%5D+lesson+feedback%3A+<lesson-id>`

## Section Checkpoint Prompt

Add this standard prompt at the end of section READMEs:

- What concept is still unclear?
- Which lesson/run command failed and why?
- What evidence would make this section easier to complete?
