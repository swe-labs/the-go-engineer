# Foundation Checkpoint

## Mission

Run the enterprise capstone successfully and explain the main moving parts before making any
meaningful change.

## Type

- flagship checkpoint

## Level

- foundation

## Prerequisites

- [10 Flagship Project](../../10-flagship-project.md)
- [rubric and checkpoint template](../../../templates/rubric-checkpoint-template.md)

## Task

1. Run the enterprise capstone in its current supported workflow.
2. Explain the role of the main runtime pieces you touched while getting it running.
3. Describe one area you would not change yet because you still need more system understanding.

## Evidence

- show the run path you used
- identify the main pieces you had to understand to get the system running
- explain the restraint decision about what you would not change yet

## Rubric

### 1. Correctness

The learner should demonstrate a correct run path for the current flagship seed.

### 2. Completeness

The learner should explain the main runtime pieces instead of only saying that it runs.

### 3. Boundary Handling

The learner should identify at least one important system seam or entrypoint.

### 4. Code Quality

The explanation should reflect real system understanding, not only directory-name repetition.

### 5. Verification Discipline

The run path and explanation should be grounded in the actual project surface.

### 6. Explanation Quality

The learner should show judgment about what is still unclear and why that matters.
