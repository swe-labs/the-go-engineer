# Iteration Checkpoint

## Mission

Choose one meaningful next improvement for the flagship seed and defend why it is the right next
step.

## Type

- flagship checkpoint

## Level

- production

## Prerequisites

- [Operations checkpoint](./operations-checkpoint.md)
- [9 Expert Layer](../../09-expert-layer.md)
- [rubric and checkpoint template](../../../templates/rubric-checkpoint-template.md)

## Task

1. Select one next improvement for the enterprise capstone.
2. Explain why it matters more than the next obvious alternatives.
3. Describe what proof would show the improvement was worth doing.

## Evidence

- name the current weakness, gap, or opportunity you are prioritizing
- explain the trade-off behind choosing it now
- define the proof surface you would use to validate the iteration

## Rubric

### 1. Correctness

The chosen improvement should fit the current state of the flagship seed.

### 2. Completeness

The learner should cover the improvement, the trade-off, and the proof plan.

### 3. Boundary Handling

The explanation should reflect how the change affects the system, not just one isolated file.

### 4. Code Quality

The proposed iteration should strengthen the system without turning into uncontrolled scope creep.

### 5. Verification Discipline

The learner should define what evidence would prove the iteration was worthwhile.

### 6. Explanation Quality

The learner should justify why this improvement wins over plausible alternatives.

## Next Step

After this checkpoint, either implement the chosen iteration or return to
[11 Code Generation](../../11-code-generation.md)
once you have enough flagship context to use generation tools deliberately.
