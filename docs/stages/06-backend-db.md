# 06 Backend & DB

## Purpose

`06 Backend & DB` is where the curriculum becomes application-shaped.

## Who This Is For

- learners who can already work across package and I/O boundaries
- developers who want to move into HTTP, persistence, and service-style code

## Mental Model

Backend work is request flow plus data flow.
This stage teaches how requests enter a system, how state is persisted, and how boundaries keep an
application understandable.

## What You Should Learn Here

- HTTP and service boundary basics
- repository-style persistence
- transactions and query discipline
- migration-aware schema work
- the first real backend-shaped milestone path

## Finish This Stage When

- you can explain a small backend flow end to end
- you understand how handlers, data access, and transactions connect
- you can reason about schema change and persistence boundaries
- you can complete the repository milestone without treating the DB layer as hidden magic
