# 03 Functions & Errors

## Purpose

`03 Functions & Errors` turns inline code into structured, reusable logic.

## Who This Is For

- learners who already understand basic values, control flow, and collections
- developers who want stronger Go-style function and error instincts

## Mental Model

Programs become maintainable when logic is split into named boundaries and failure is handled
explicitly.

## What You Should Learn Here

- function boundaries and decomposition
- parameters and return values
- multiple-return workflows
- errors as values
- validation and orchestration habits that prepare you for larger designs

## Current Source Content

- [01-foundations/05-functions-and-errors](../../01-foundations/05-functions-and-errors/)

## Finish This Stage When

- you can turn inline logic into clear functions
- you handle failures with explicit `(value, error)` flow instead of guesses
- you can validate inputs and explain why failure should be surfaced early
- you can complete the order-summary milestone without hidden magic
