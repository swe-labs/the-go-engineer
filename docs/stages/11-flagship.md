# 11 Flagship

## Purpose

`11 Flagship` is the long-running proof surface for the curriculum.

## Who This Is For

- learners who want one product spine that integrates many stages together
- developers who want a portfolio-quality proof path inside the repo

## Mental Model

A flagship is where backend, concurrency, testing, architecture, operations, and automation start
to interact inside one evolving system.

## What You Should Learn Here

- staged feature growth instead of one giant final dump
- architectural and operational trade-offs inside a single system
- checkpoint-driven iteration
- responsible use of code generation in the context of a real system
