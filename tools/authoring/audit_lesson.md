# Audit Lesson

## Purpose

Audit a lesson for zero-magic quality, README heading order, runnable code, test value, starter/solution alignment, diagrams, common mistakes, debugging guidance, production context, and next-step accuracy.

## Required process

1. Read the relevant metadata record.
2. Check prerequisites, targets, and contracts.
3. Inspect existing learner-facing files when present.
4. Write or revise content using the current architecture.
5. Run focused validation.
6. Run the strict release gate before claiming completion.

## Completion evidence

- Metadata references resolve.
- Files exist in the canonical typed path.
- README quality contract is satisfied.
- Code and tests pass where required.
- Project or assessment evidence is complete where applicable.
- No forbidden placeholder language remains.
